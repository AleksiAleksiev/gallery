# Gallery

An interactive atlas of Western painting, 1150 to the present, with a walkable 3D museum room for every artist. Live at **[gallery.lokset.dev](https://gallery.lokset.dev)**.

- **The atlas** (`/`) — an engraved-plate strata chart of 19 periods and 95 painters. Zoom, pan, filter by period or artist, and open a placard for any painter.
- **The museum** (`/museum/<artist>`) — a procedurally built gallery hall hung with that artist's paintings (WASD + mouse look, click a painting to inspect it up close). Enter through a placard, or walk back out through the door to return to the atlas.

All content — bios, dates, painting metadata, images, attributions — is drawn from Wikipedia, Wikidata, and Wikimedia Commons. The only hand-curated input is the seed list in [`pipeline/seed-data.ts`](pipeline/seed-data.ts): which periods, which artists, which must-include paintings, plus a few editorial overrides where the source data is wrong.

## Repo layout

| Path | What it is |
|---|---|
| `src/app`, `src/components`, `src/lib` | Next.js app (static export): the timeline atlas and the React Three Fiber museum |
| `pipeline/` | Five-stage seed pipeline (see below) |
| `data/` | Pipeline workspace: SQLite DB, API cache, exported JSON — **not committed** |
| `public/images/` | Processed painting/portrait images (~1.9 GB) — **not committed** |
| `scripts/deploy.sh` | Build + rsync to the hosting VM |

The heavy artifacts aren't in git. Everything under `data/` and `public/images/` is regenerated by the pipeline; the deployed site gets its images via rsync, not via the repo.

## Regenerating the data

```bash
npm install
npm run seed          # runs all five stages; hours on first run, cached after
```

Stages (each runnable individually as `npm run seed:artists`, `seed:paintings`, `seed:images`, `seed:validate`, `seed:export`):

1. **artists** — resolves each seed artist to Wikidata, pulls dates, bios, portraits.
2. **paintings** — SPARQL per artist for notable paintings (sitelink-ranked, must-includes matched), with normalized dimensions and dates.
3. **images** — downloads and resizes Commons images into tiers; artists marked `copyrighted` get placeholder canvases with full metadata.
4. **validate** — cross-checks the set and reports gaps.
5. **export** — writes `data/json/` (consumed at build time) with cache-busting image URLs.

Every API response is disk-cached under `data/cache/`, and requests are rate-limited with a descriptive User-Agent, so re-runs are fast and polite to the Wikimedia APIs.

## Developing and deploying

```bash
npm run dev           # needs data/json + public/images (run the seed first)
npm run build         # static export to out/
npm run deploy        # build + rsync out/ to the VM (see scripts/deploy.sh)
```

The deploy script is specific to this site's host (an Oracle VM behind Caddy); adapt it if you're hosting elsewhere. Any static file server works — there is no backend.
